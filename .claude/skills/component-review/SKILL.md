指定されたコンポーネントファイルを以下の観点でレビューしろ。

## チェック項目

### TypeScript
- Props の型定義が正しくあるか
- `any` を使っていないか
- 型推論で問題ないところに冗長な型注釈がないか

### コンポーネント設計
- 150行を超えていないか（超えてたら分割提案を出せ）
- `'use client'` が必要な場所に付いているか、不要な場所に付いていないか
- 再レンダリングの無駄がないか（`useMemo` / `useCallback` の適切な使用）

### Tailwind CSS
- レスポンシブ対応（`sm:` / `md:` / `lg:`）が必要な箇所に適用されているか
- マジックナンバーの `w-[〇〇px]` が多用されていないか

### アクセシビリティ
- `img` に `alt` があるか
- インタラクティブ要素に `aria-label` が必要な場合に付いているか
- キーボード操作で使えるか

## 出力フォーマット
問題なし → 「LGTM」とだけ返せ。
問題あり → 以下の形式で出せ：

**[ファイル名:行番号]**
- 問題: （何が問題か）
- 修正案: （どう直すか）

ファイルへの書き込みは絶対にするな。指摘のみ。
