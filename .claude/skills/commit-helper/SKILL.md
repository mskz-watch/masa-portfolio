以下の手順でコミットを実行しろ。

## 手順

1. `git diff --staged` で staged の変更内容を確認しろ
2. staged が空なら `git status` で変更ファイルを確認しろ
3. 変更内容に基づいてコミットメッセージを日本語で作れ

## コミットメッセージのルール
- prefix必須: `feat` / `fix` / `refactor` / `style` / `docs` / `chore`
- 1行目: prefix + 変更の要約（40字以内）
- 例: `feat: Worksページのカードコンポーネント追加`
- 例: `fix: SPでナビゲーションが崩れる問題を修正`
- 例: `style: HeroセクションのTypographyを調整`

## 注意
- 複数の独立した変更がある場合は「まとめてコミットしないほうがいい」と指摘しろ
- `.env` 系のファイルが staged に含まれていたら即座に警告して止まれ
